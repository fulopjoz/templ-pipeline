Function Reference

Orchestration & Alignment

  • transform_ligand(): Loads mobile protein, filters amino acids, validates chains, extracts CA atoms, runs hierarchical alignment (homologs → sequence → 3Di → centroid), transforms ligand coords, validates bonds, and sets metadata.
  • _validate_and_select_chains(): Validates requested chains or falls back to all available chains.
  • Constants: MIN_CA_ATOMS_FOR_ALIGNMENT=3, MIN_ANCHOR_RESIDUES=15, BLOSUM_GAP_PENALTY=-10.

Alignment Levels

  • superimpose_homologs(): For similar-length sequences; extracts anchors and computes RMSD.
  • _align_with_biotite_sequence(): Converts structures to sequences, aligns with BLOSUM62, extracts anchors, superimposes, returns transformation and RMSD.
  • _align_with_3di_structural(): Uses structural alphabet when available; aligns and returns transformation and RMSD.
  • _align_with_centroid_fallback(): Uses first N CA atoms when others fail.

Data & Templates

  • load_reference_protein(): Loads reference PDB via biotite.
  • load_target_data(): Builds target RDKit molecule from SMILES and adds hydrogens.
  • filter_templates_by_ca_rmsd(): Filters by CA RMSD threshold.
  • get_templates_with_progressive_fallback(): Progressive CA RMSD thresholds with central-atom fallback selection.
  • load_template_molecules_standardized(): Loads templates via shared cache, with optional SMILES exclusion.
  • pdb_path() / ligand_path(): Resolve file paths in common locations.
