Function Reference Table

Data Loading Stage

  • load_reference_protein(): Loads reference PDB structure
  • load_target_data(): Creates target molecule from SMILES
  • filter_amino_acids(): Extracts protein residues only
  • get_chains(): Identifies available protein chains

Chain Validation Stage

  • _validate_and_select_chains(): Validates binding site chains
  • Falls back to all available chains if needed
  • Ensures minimum 3 CA atoms for alignment

Level 1: Homologous Alignment

  • superimpose_homologs(): Uses BLOSUM62 matrix
  • Requires sequence length similarity
  • Minimum 15 anchor residues required
  • Gap penalty: -10

Level 2: Sequence Alignment

  • _align_with_biotite_sequence(): Optimal sequence alignment
  • to_sequence(): Converts structures to sequences
  • align_optimal(): Performs optimal alignment
  • _rmsd_from_alignment(): Extracts anchors and calculates RMSD

Level 3: 3Di Structural

  • _align_with_3di_structural(): Uses 3Di structural alphabet
  • Requires STRUCTURAL_ALPHABET_AVAILABLE
  • For remote homologs with poor sequence similarity
  • Uses 3Di substitution matrix

Level 4: Centroid Fallback

  • _align_with_centroid_fallback(): Simple centroid alignment
  • superimpose(): Basic structural superimposition
  • Uses first N atoms from both structures
  • Final fallback when all other methods fail

Quality Assessment

  • rmsd(): Calculates CA RMSD
  • filter_templates_by_ca_rmsd(): Filters by threshold
  • Progressive fallback: 10Å, 15Å, 20Å
  • Bond length validation: 0.5-3.0Å range

Coordinate Transformation

  • transformation.apply(): Bulk coordinate transformation
  • Point3D coordinate setting for each atom
  • Bond length validation ensures geometry
  • Metadata: ca_rmsd, template_pid, alignment_method
